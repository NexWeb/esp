Working with the ESP Source Code

ESP releases are available as either a binary or source code distribution. It is your choice which distribution you choose, but the source code distribution has several advantages:

  • You can recompile ESP to optimize it for your specific system environment.
  • You can reconfigure ESP via configuration options to select the specific feature set you require.
  • You can minimize the memory footprint and decrease the securable surface area.

Building ESP from Source

The ESP source distribution contains all the required source files, headers, tools, and test framework to reconfigure, build, and verify ESP. For details, see the building from source instructions.

Porting the Source to a New Platform

ESP currently runs on a wide variety of operating platforms including: FreeBSD, Linux, MAC OSX and Windows.

Accessing the Source Code

With each ESP release, a complete source code snapshot is provided. You can get the latest source code from the download site or you can access the source code repository.

Source Repository

The ESP source code is hosted in a Git repository. We provide open read-only access. Write access is available to regular contributing developers. To check out the source use this command:

$ git clone http://github.com/embedthis/esp

You can also view the repository at https://github.com/embedthis/esp. This is also a convenient way to get the source code.

Repository Branches

The repository has three primary branches for development and release: master, lts and dev.

Master Branch

The master branch in the repository is used for released versions. The HEAD of the branch contains the most recent release of the software. This branch is regarded as stable and ready for production use.

LTS Branch

The lts branch in the repository is used for the LTS release versions. The HEAD of the branch contains the most recent LTS release of the software. This branch is regarded as stable and ready for production use.

Dev Branch

The Dev branch in the repository is used for ongoing developement. New features and test fixes are pushed to this repository on a regular basis. It is not as bug-free as the Master branch and should not be used for production.
