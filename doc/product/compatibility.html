Compatibility

ESP is provided in two supported versions:

  • ESP Long Term Support (LTS) — The goal of LTS versions is stability and compatibility. Bugs and security issues will be fixed, but no feature updates will be issued for LTS versions. ESP 4.5.X is head of the LTS stream and it will be maintained with security updates and fixes until the end of 2015.
  • ESP Development Versions (DEV) — The goal of DEV versions is ongoing feature development. New features will be issued for DEV versions and APIs will evolve. ESP 5 is the next DEV version and it will be released in Q1 2014.

API Compatibility

In the development stream (ESP 5.X) the ESP APIs may evolve, and are thus subject to change between minor releases. The ESP documentation provides a Stability Classification for each API group that reflects their stability with respect to forward changes.

Migrating to ESP 5

ESP 4 and 5 are highly compatible. Code written for ESP 4 should migrate to ESP 5 with little or no changes required.

If you are migrating from ESP 2 or 3, you should regard ESP 5.X as a next generation web server. You can migrate code from ESP 2 or 3 to ESP 5, however, you should anticipate significant refactoring work in the transition.
