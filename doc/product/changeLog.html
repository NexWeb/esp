ESP Change Log

Brief ESP change log of the the major features and changes for each release.

Note to security researchers:

Please do not mine this list and create security advisories. Please see the ESP Issue Database for the official product issue list and for comprehensive CVE dispositions.

Changes in 5.1.0

Minor Features

  • Simplify log and trace output to the console via --trace and --log switches [issue 21].
  • Add unit tests using TestMe [issue 22].

Fixes

  • Fix race condition with mprWaitForEvent [issue 23].
  • Fix JSON config file parsing [issue 24].

Changes in 5.0.0

Major Changes

  • ESP extracted from Appweb as a stand-alone product [issue 2].
  • Simplified ESP serving. ESP can serve applications without requiring an appweb.conf. [issue 3].
  • ESP and HTTP can be fully Configured via package.json. [issue 5].
  • Enhanced ESP to support progressive install via Pak packages. [issue 6].
  • Enhanced ESP to install, manage and uninstall Pak packages [issue 1].
  • Split error log and request log [issue 14].

Incompatible Changes

  • ESP Package.json format is changed for many properties. Regenerate the application to get a new package.json file. The following package.json properties have been moved or modified:

    • A top level app collection contains all non-CommonJS properties.
    • client-scripts has been renamed to app.client.scripts.
    • Most properties under esp have been moved to the app.http collection.
    • The esp property has been renamed app.esp.
    • The mode and modes properties have been moved under the app collection.
    • The auth property is renamed app.http.auth.store.

    See the esp-configuration Sample for details of all supported configuration file properties.

  • ESP Angular applications have a different Esp.config property layout. Upgrade all relevant paks.
  • With separating the error log and request log, the mprLog and mprError APIs have revised signatures.

Fixes since rc1

  • Auth roles and abilities. [issue 15].
  • Make.bat reference to old bit program. [issue 16].
  • Parsing multiple and corrupted http cookies. [issue 17].
  • Rendering fix.css before all.less [issue 18].
  • Compiling outside top directory fails to warn [issue 19].
  • Fix printf format strings. Enable compiler printf format checking. [issue 20].

Fixes since rc0

  • esp --name being ignored. [issue 8].
  • Disable XSRF by default in routes. Enable in ESP route sets. [issue 9].
  • Set HttpRoute.prefix and HttpRoute.serverPrefix at all times. [issue 10].
  • Enhanced ESP to create and manage users and roles [issue 11].
  • Update ESP to fix client.mappings timeouts [issue 12].
  • ESP upgrade should not overwrite overwrite files by default. [issue 13].
